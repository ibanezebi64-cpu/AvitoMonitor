# Architecture and Plan: Avito VK Monitor Bot

## 1. Архитектура проекта

Проект представляет собой backend-приложение на Node.js (TypeScript), которое будет работать как VK-бот и демон для мониторинга Avito.

### 1.1 Основные компоненты
- **VK Bot Interface**: Использование библиотеки `vk-io` для работы с VK API (Long Poll). 
- **Database (БД)**: `SQLite` (с использованием `better-sqlite3` для легковесности) для хранения пользователей, их выбранных категорий, настроек фильтрации и истории отправленных объявлений (чтобы не присылать дубликаты).
- **Scraper / Parser (Парсер Avito)**: Модуль, который периодически делает запросы к Avito.
  - *Проблема блокировок*: Avito использует сильную защиту (Antibot/Cloudflare).
  - *Решения*:
    - Поддержка прокси-сервера (HTTPS) через `.env` (PROXY_URL).
    - Динамические интервалы (от 12 до 18 минут) между глобальными проходами и точечные паузы 15-30 секунд между категориями.
    - Динамичная генерация User-Agent, платформы и заголовков `Sec-Ch-Ua` для скрытия "ботности" парсера.
    - Автоматическая заморозка работы парсера на 60 минут при получении 403 или 429 кода (Бан IP).
- **Scheduler (Планировщик)**: `node-cron` для периодического запуска задачи парсинга (например, раз в 5-10 минут).
- **Web Server**: Minimal Express сервер (порт 3000) для поддержания приложения в рабочем состоянии (на серверах/VPS/Cloud Run) и базового health-check.

### 1.2 Структура Данных (Сущности)
1. **User**: 
   - `vk_id` (PK, Уникальный идентификатор ВК)
   - `is_active` (boolean, включен ли мониторинг)
   - `state` (string, текущее состояние пользователя в меню бота)
2. **Category**:
   - `id` (PK)
   - `user_id` (FK -> User)
   - `category_code` (Идентификатор категории из дерева, например `phones`)
   - `title` (Название, например "Мобильные телефоны")
3. **Filter**:
   - `id` (PK)
   - `category_id` (FK -> Category)
   - `min_price`, `max_price` (int)
   - `condition` (string, 'new', 'used', 'all')
   - `query` (строка поиска)
4. **SeenAd (Просмотренные объявления)**:
   - `id` (PK)
   - `avito_id` (Идентификатор объявления)
   - `user_id` (FK -> User)
   - `created_at` (Дата отправки уведомления)

### 1.3 Логика работы бота
1. `/start` -> Регистрация пользователя в БД -> Отправка приветствия + Inline клавиатура (Главное меню).
2. Навигация внутри бота: обработка message payload и payload кнопок, переход по меню (смена `state`).
3. Парсер: Cron каждые несколько минут обходит всех пользователей с `is_active = true`. Берет их категории и фильтры, запрашивает новые объявления.
4. Отправка: Новые объявления (извлеченные картинки, текст) отправляются пользователю. Записывается отбивка в `SeenAd`.

## 2. План Реализации (Roadmap)

- [ ] **Этап 1: Инициализация и Базовая структура**
  - Подготовка бэкенд сервера (порт 3000).
- [ ] **Этап 2: Подключение Базы Данных**
  - Установка и инициализация `better-sqlite3`.
  - Написание SQL миграций для создания таблиц по структуре выше.
- [ ] **Этап 3: Подключение VK Bot Framework**
  - Установка `vk-io`.
  - Подключение к Long Poll.
  - Настройка обработки базовых команд (start, главное меню).
- [ ] **Этап 4: Разработка интерфейса VK (Клавиатуры и Сценарии)**
  - Реализация главного меню.
  - Добавление статического дерева категорий.
  - Реализация выбора категорий, настройки фильтров в них.
- [ ] **Этап 5: Разработка Парсера Avito**
  - Реализация парсинга Axios + Cheerio. Для обхода защиты добавлены:
    - Использование динамических интервалов опроса (например, от 12 до 18 минут) для имитации человеческого поведения.
    - Ротация User-Agent и подстановка правильных заголовков браузера (SEC-CH-UA).
    - Поддержка прокси (HttxProxyAgent) через переменные окружения (.env).
    - Блокировка на 60 минут (Cooldown) при попадании под капчу/CloudFlare.
- [ ] **Этап 6: Сборка всех компонентов (Планировщик)**
  - Периодичный опрос Avito и рассылка уведомлений.
- [ ] **Этап 7: Админ-панель**
  - Команда для админа со статистикой.
